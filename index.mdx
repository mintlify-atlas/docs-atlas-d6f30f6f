WP SSR Framework turns WordPress into a headless CMS powerhouse. It intercepts every request at the PHP level, routes it through typed controllers, renders a React 19 + Vite SPA with server-injected data, and optionally snapshots pages to static HTML for instant bot delivery — all without touching a WordPress theme.

## How it works

<Steps>
  <Step title="WordPress receives a request">
    Bedrock boots WordPress. The `wp-ssr` mu-plugin registers the kernel and all service providers on `wp_loaded`.
  </Step>
  <Step title="The web router matches the URL">
    `WebRouter` compares the request URI against your registered routes and CPT paths. Matching requests are handed to a PHP controller — WordPress theme rendering never runs.
  </Step>
  <Step title="The controller builds the page payload">
    Your controller queries custom post types via `EloquentCPT`, assembles SEO metadata, and passes a `__wp_data__` JSON payload to the React shell template.
  </Step>
  <Step title="React mounts and hydrates">
    The browser receives an HTML shell containing `window.__wp_data__` and `window.__wp_view__`. The Vite-built SPA boots, resolves the correct page component, and mounts — no full reload required for subsequent navigations.
  </Step>
  <Step title="Bots get static HTML">
    On publish, the SSG service calls an external headless-browser API to snapshot the rendered page and writes it to `web/app/cache/client/static/`. Detected bots receive this file directly with an `X-SSG-Cache: HIT` header.
  </Step>
</Steps>
